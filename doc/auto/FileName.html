FileName

Represents a filename in the host OS. These are created from a string and allow access to the elements of the name.

Win32 FileNames are divided into four parts: Root, Directory, BaseName, and Extension. Any or all of these parts can be null. Some examples: 


readme.txt          "", "", "readme", ".txt"       Relative (win32), Absolute (POSIX)
/readme.txt         "/", "", "readme", ".txt"      Relative
d:\readme.txt       "d:\", "", "readme", ".txt"    Absolute
d:readme.txt        "d:", "", "readme", ".txt"     Relative
d:\src/readme       "d:\", "src/", "readme", ""    Absolute
\\host\c\src\rm     "\\", "host\c\src\", "rm", ""  Absolute
D:\                 "D:\", "", "", ""              Absolute
D:\src.d            "D:\", "", "src", ".d"         Absolute
D:\src.d\           "D:\", "src.d\", "", ""        Absolute
D:\src.d\*.*        "D:\", "src.d\", "*", ".*"     Absolute, Pattern
D:\src\..           "D:\", "src\", "..", ""        Absolute
D:\src\.            "D:\", "src\", ".", ""         Absolute
D:\src\.foo         "D:\", "src\", "", ".foo"      Absolute

On UNIX systems, the root is always either "/" or "".
