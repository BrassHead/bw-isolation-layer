BW

An application framework

Scope

What facilities are needed? The full intended scope is illustrated by this diagram which was the start of the project.


BW Overview

Thus the "packages" of BW facilities are:

  • Language
  • Graphics
  • Files and directories
  • Resources and preferences
  • Processes and threads
  • Persistent storage
  • Network access
  • Distributed objects
  • File format experts
  • Scripting
  • Help system
  • Localization

This is a big project and it will not succeed by pie in the sky dreaming or big bang implementation. Thus, we expect to drive the implementation by doing real applications and working to provide the necessary isolation layer as required. These applications will get increasingly sophisticated and thus require an increasingly sophisticated isolation layer.

What things are not in the scope?

The Makefile
The mechanism to build BW applications is not defined as part of BW and the makefile is not considered part of the "code" that has long duration. Note that in order to have standard makefiles, you need to define a shell, a set of standard utilities, and standard options for development tools and then provide isolation tools for all of these on the target platform. This is a huge effort with a small return. Furthermore, most modern environments have a lot of automation for makefile creation, so this is not, in practice, a huge portability problem.
Installation
BW applications are installed in some platform dependent way.
